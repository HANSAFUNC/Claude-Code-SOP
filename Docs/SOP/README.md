## 🔄 流程总览

```
需求输入 → 架构设计 → 并行开发 → 测试验证 → 代码审查 → 合并发布
   ↓          ↓          ↓          ↓          ↓          ↓
 PRD+Design  Gate 1    Code+Test  Gate 2     Gate 3     Release
```

### 阶段说明

| 阶段 | 负责人 | 参与人 | 产出物 | 准出条件 |
|------|--------|--------|--------|----------|
| 需求输入 | PM | 全员 | PRD、Design | 文档完整 |
| 架构设计 | Architect | 全员 | 架构文档 | 评审通过 |
| 并行开发 | Specialist | QA | 代码 + 测试 | 自测通过 |
| 测试验证 | QA | Specialist | 覆盖率报告 | 覆盖率≥80% |
| 代码审查 | Architect | 全员 | 审查报告 | 问题修复 |
| 合并发布 | Team Lead | 全员 | Release | 发布成功 |

---

## 🎯 质量门禁

| 门禁 | 阶段 | 检查项 |
|------|------|--------|
| **Gate 1** | 架构设计后 | 架构合理、模型完整、API 清晰 |
| **Gate 2** | 开发完成后 | 测试通过、覆盖率≥80% |
| **Gate 3** | 审查完成后 | 问题解决、Architect 批准 |
