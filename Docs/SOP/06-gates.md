# 质量关卡 (Quality Gates)

**版本:** 1.0
**日期:** 2026-03-20

---

## 1. 质量关卡概览

```
需求输入 → Gate 1 → 架构设计 → Gate 2 → 并行开发 → Gate 3 → 测试验证 → Gate 4 → 代码审查 → Gate 5 → 合并发布
            ↓                    ↓                    ↓                    ↓                    ↓
        需求评审             架构评审             自测通过            覆盖率达标            审查批准
```

---

## 2. Gate 1: 需求评审

**触发条件:** PRD 和 Design 上传完成

**检查项:**
| 检查项 | 标准 | 责任人 |
|--------|------|--------|
| PRD 完整性 | 需求描述清晰完整 | PM |
| Design 完整性 | 设计稿清晰可用 | 设计师 |
| 技术可行性 | 技术可实现，无明显风险 | Architect |
| 需求理解 | 全员理解一致 | 全员 |

**准出条件:**
- [ ] PRD 评审通过
- [ ] Design 评审通过
- [ ] 技术可行性确认
- [ ] 全员理解一致

---

## 3. Gate 2: 架构评审

**触发条件:** 架构设计文档完成

**检查项:**
| 检查项 | 标准 | 责任人 |
|--------|------|--------|
| 架构设计 | 架构合理、清晰 | Architect |
| 数据模型 | 模型定义完整 | Architect |
| API 设计 | 接口设计合理 | Architect |
| 技术选型 | 选型适当 | 全员 |

**准出条件:**
- [ ] ArchitectureDesign.md 完成
- [ ] 数据模型定义完成
- [ ] API 接口设计完成
- [ ] 全员评审通过

---

## 4. Gate 3: 自测通过

**触发条件:** 代码开发完成

**检查项:**
| 检查项 | 标准 | 责任人 |
|--------|------|--------|
| 编译检查 | 无编译错误/警告 | Platform Spec |
| 单元测试 | 所有单元测试通过 | Platform Spec |
| 代码规范 | 符合编码规范 | Platform Spec |

**准出条件:**
- [ ] 代码编译通过
- [ ] 所有测试通过
- [ ] 无编译警告

---

## 5. Gate 4: 测试验证

**触发条件:** 自测通过

**检查项:**
| 检查项 | 标准 | 责任人 |
|--------|------|--------|
| 单元测试覆盖率 | ≥80% | QA |
| UI 测试 | 关键路径通过 | QA |
| 性能测试 | 无明显性能问题 | QA |

**准出条件:**
- [ ] 单元测试覆盖率 ≥ 80%
- [ ] UI 测试关键路径通过
- [ ] 测试报告输出

---

## 6. Gate 5: 代码审查

**触发条件:** 测试验证通过

**检查项:**
| 检查项 | 标准 | 责任人 |
|--------|------|--------|
| 代码审查 | 初审通过 | Platform Spec |
| 架构审查 | 复审通过 | Architect |
| 问题修复 | 所有问题解决 | Platform Spec |

**准出条件:**
- [ ] 代码审查通过
- [ ] 架构审查通过
- [ ] 所有问题解决

---

## 7. Gate 6: 合并发布

**触发条件:** 代码审查通过

**检查项:**
| 检查项 | 标准 | 责任人 |
|--------|------|--------|
| 代码合并 | 无冲突 | Team Lead |
| 版本发布 | Release 创建 | Team Lead |
| 文档更新 | Changelog 更新 | Team Lead |

**准出条件:**
- [ ] 代码成功合并
- [ ] Release 创建成功
- [ ] Changelog 已更新

---

## 8. 质量指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 测试覆盖率 | ≥80% | 代码覆盖率 |
| 审查响应时间 | <24 小时 | 从提交到审查 |
| 缺陷率 | <5% | 合并后发现问题 |
| 构建成功率 | >95% | CI/CD 构建成功 |
