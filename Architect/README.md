# 架构设计目录

> 存放项目特定的架构设计文档

---

## 📁 目录说明

此目录用于存放项目特定的架构设计文档，由 Architect 角色负责编写和维护。

### 文件命名规范

```
ArchitectureDesign.md    # 整体架构设计
APIDesign.md             # API 接口设计
DataModels.md            # 数据模型定义 (可选)
```

---

## 📝 使用说明

**新项目接入时:**

1. 根据 Phase 1 设计阶段产出，在此目录创建架构设计文档
2. 参考 SOP 流程中 Phase 0 规范探索步骤
3. 从现有代码提取架构模式并定义

**文档内容建议:**

```markdown
# 架构设计

## 1. 整体架构
- 架构模式 (MVC/MVVM/Clean/etc)
- 模块划分
- 依赖关系

## 2. 数据模型
- 核心数据结构
- 数据关系定义

## 3. API 设计
- API 端点定义
- 请求/响应格式

## 4. 技术选型
- 框架选择
- 依赖库

## 5. 目录结构
- 项目目录组织
- 文件命名约定
```
